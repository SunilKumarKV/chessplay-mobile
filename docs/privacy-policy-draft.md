# ChessPlay Mobile Privacy Policy Draft

This is a draft for legal/product review and should not be published without replacing placeholders and confirming actual backend behavior.

## Account Data

ChessPlay accounts may collect username, email address, password credentials handled by the backend, profile details, country, bio, avatar, rating, and account status.

## Gameplay Data

ChessPlay stores gameplay data such as moves, game results, ratings, game history, puzzle attempts, puzzle progress, and live room metadata needed to operate multiplayer chess.

## Messages And Social Data

ChessPlay may store friend relationships, friend requests, private messages, room chat, community content, profile information, and moderation-related metadata.

## Analytics Placeholder

Analytics and crash reporting are not finalized in the mobile app. If analytics are enabled later, disclose the provider, data collected, retention, opt-out options, and whether device identifiers are used.

## Push Notification Placeholder

The mobile app can request notification permission after login and collect an Expo push token on physical devices. Push delivery is not enabled until the backend stores tokens and sends notifications. If enabled, ChessPlay may store device push tokens to send account, game, message, friend, or tournament notifications.

## Native Device Features Placeholder

ChessPlay mobile may use device features such as biometric authentication for local app unlock, native sharing for room/profile/result links, and deep links for room invites, profiles, password reset, and email verification. Biometric data is handled by the operating system and is not sent to ChessPlay servers.
